Yeah, What's up Ember.JS ?

Mise à jour du Routeur

  • transitionTo et les "transitions par URL" se comporte maintenant de la même façon (déclenchement de hooks par exemple)
  • Les 2 méthodes attendent maintenant la résolution de promesse renvoyées par le model avant de finir la transition

Manipulation de transition via le model

  • 2 nouvelles méthodes : beforeModel, afterModel en plus de model dans le routeur pour manipuler les transitions
  • Chacune de ces 3 méthodes peut mettre la transition en attente en renvoyant une promesse

App.MyRoute = Em.Route.extend({
  beforeModel: function(transition) {
        if(!this.controllerFor('login').get('isLogged')) {
            this.transitionTo('login');
        }
  }
});

Manipulation de transition via les évènements

Ajout d'un évènement willTransition permettant de rediriger, annuler ou décorer la transition en cours


App.MyRoute = Em.Route.extend({
  actions: {
     willTransition: function(transition) {
        if(false) {
            transition.abort();
        }
     }
  }
});

Gestion d'erreur

En cas de promesse rejeté, ou d'erreur lancé depuis les méthodes beforeModel,model, ou afterModel un evenement erreur est lancé


App.MyRoute = Em.Route.extend({
  model: function() {
        return $.getJSON('invalidURL'); // $.getJSON renvoit une promesse
  },
  actions: {
     error: function(reason, transition) {
        console.log(reason.message);
        if(reason.status == 401) {
            this.transitionTo('login'); // redirection
        } else {
            return true; //bubble de l'erreur
        }
     }
  }
});

Rejouabilité des transitions

  • Les objets transitions peuvent être sauvegardé et rejoué par la suite via la méthode retry
  • Un cas d'usage classique : l'authentification ; vous etes redirigé vers la bonne page après authentification

Évènements

  • Déclaration des actions dans un attribut actions des controllers et des routes
  • actions remplace donc events dans les routes

//dans un template
<button {{action 'save'}}>save</button>

//Dans le code JS
App.MyController = Em.Controller.extend({
    actions: {
        save: function() {
            // Code métier
            return true; //bubbling
        }
    }
});

App.MyRoute = Em.Route.extend({
    actions: {
        save: function() {
            // l'évenement bubble jusqu'à la route
        }
    }
});

Component

  • Un "component" est un bout de code réutilisable généralement constitué d'un template et d'une vue
  • Les "components" d'Ember se veulent être le plus proche possible des "Web Components" définis par le W3C

<script type="text/x-handlebars" id="components/blog-post">
  <h1>Blog Post : {{title}}</h1>
  <p>{{yield}}</p>
</script>


//Dans un autre template :
{{#each}}
  {{#blog-post title=title}}
        {{body}}
  {{/blog-post}}
{{/each}}

Mise à jour de sécurité

  • Ember en version antérieur à la RC7 était exposé à des attaques de type XSS
  • Des versions *.1 sont sorties rapidement pour corriger cette faille (RC6.1, RC5.1, RC4.1, etc.)
  • Si vous utilisez une version non patchée, pensez à la mettre à jour !

Divers

  • La run-loop est maintenant séparé dans son propre module "Backburner"
  • Les observers ne sont plus déclenchés à la construction de la classe
  • Si vous utilisez une version non patchée, pensez à la mettre à jour !
